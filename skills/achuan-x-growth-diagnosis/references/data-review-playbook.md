# 数据复盘作战规则

来源说明：本文件为开源版蒸馏规则，来自 X 数据作战计划类材料的通用化提炼；不保留原包品牌、PDF 外壳、Claude 专用路径或私有素材。

## 使用场景

- 用户上传 X Analytics CSV，要求分析账号数据。
- 用户问哪类内容带涨粉、哪类内容无用功。
- 用户要求 7 天复盘、30 天计划或季度复盘。
- 用户带历史账号成长档案 JSON，要求对比上期执行效果。

## 公理

- 数据复盘的目的不是做漂亮报告，而是找出下一期最该加码和最该砍掉的动作。
- 单条爆款只能作为线索，连续出现的内容类型、钩子结构和涨粉效率才是策略依据。
- “曝光高”不等于“涨粉强”；应同时看曝光、关注增长、收藏、互动和主页承接。
- 回复是必要互动，但回复占比过高时，可能挤压原创内容和长文生产。

## 输入清单

| 输入 | 用途 | 缺失时 |
|---|---|---|
| X Analytics CSV | 计算内容类型、曝光、涨粉、互动 | 先让用户补数据或改走手动诊断 |
| 当前粉丝数 | 判断阶段和 KPI 基准 | 询问用户 |
| handle / 昵称 / bio | 判断账号方向和主页承接 | 可先留空，但报告标注不完整 |
| 账号方向 | 设置分类关键词和内容支柱 | 先从 bio 和 TOP 帖推断，再让用户确认 |
| 账号成长档案 JSON | 对比历史和执行率 | 没有则做首次基准 |
| 上期执行自评 | 判断数据偏差原因 | 没有则只做数据侧复盘 |

## 模式判断

| 模式 | 触发条件 | 输出重点 |
|---|---|---|
| 首次基准 | 有 CSV，无账号成长档案 | 基准诊断、爆款公式、30 天假设计划、建议保存成长档案 |
| 7 天复盘 | 有 CSV + 成长档案，距上次复盘较近 | 执行评分、KPI 对比、计划偏差、接下来 7 天微调 |
| 季度复盘 | 有 CSV + 成长档案，跨越多期或约 60 天以上 | 阶段判断、增长轨迹、爆款规律进化、内容疲劳检测 |

## 字段映射

| 标准字段 | 常见列名 | 用途 |
|---|---|---|
| post_text | Post text / 帖文内容 | 分类、提炼钩子、识别回复 |
| impressions | Impressions / 展示次数 | 曝光效率 |
| likes | Likes / 点赞数 | 互动质量 |
| engagements | Engagements / 互动次数 | 互动率 |
| bookmarks | Bookmarks / 收藏数 | 工具价值和信任信号 |
| shares | Shares / Reposts / 转发数 | 传播系数 |
| new_follows | New follows / 新增关注 | 转粉效率 |
| replies | Replies / 回复数 | 互动结构 |
| date | Date / 日期 | 趋势、时段和周期复盘 |

如果列名不匹配，先列出 CSV 字段并让用户确认映射，不要硬猜。

## 分析维度

1. 基础盘：总帖数、总曝光、均值曝光、中位数曝光、最高单帖曝光、新增关注。
2. 内容类型：短推、长文/thread、链接、回复、引用、图文、用户自定义主题。
3. 涨粉效率：每类内容的新增关注、涨粉/万曝、收藏/曝光。
4. 头部集中度：TOP 10 帖贡献了多少曝光和新增关注。
5. 回复占比：回复数量 / 总帖数；同时看回复带来的新增关注。
6. 爆款公式：从 TOP 涨粉帖和 TOP 曝光帖提炼共同结构。
7. 内容疲劳：同一类型连续多期均值下降时，降低比例并测试新角度。

## 执行评分

执行评分用于复盘上一期计划，不用于责备用户。

| 等级 | 执行率 | 判断 |
|---|---:|---|
| A | >= 85% | 严格执行，数据高度可信，可直接校准下一期 |
| B | 60%-84% | 基本执行，数据可参考，找拖后腿环节 |
| C | 30%-59% | 执行不稳定，先缩小计划颗粒度 |
| D | < 30% | 计划失配，先重做可持续最低执行量 |

参考权重：长文执行率 40%，短推执行率 40%，回复比例达标 20%。如果账号阶段不适合高频长文或短推，先调整目标再评分。

## 诊断信号

| 信号 | 解释 | 路由 |
|---|---|---|
| 长文少但涨粉效率高 | 信任内容不足 | `/achuan-x-content-rhythm` |
| 短推曝光高但关注少 | 破圈有余，关注理由不足 | `/achuan-x-profile` 或 `/achuan-x-content-system` |
| 回复占比高且转粉低 | 时间被低效互动吞掉 | `/achuan-x-reply-growth` |
| 收藏高于点赞 | 工具价值强，可加码教程/清单 | `/achuan-x-content-system` |
| 连续两期均值下降 | 可能内容疲劳 | `/achuan-x-content-system` |
| 突然断崖式下降 | 先排除风险或可见度异常 | `/achuan-x-risk-recovery` |

## 输出契约

CSV 复盘必须输出：

- 当前模式：首次基准 / 7 天复盘 / 季度复盘。
- 三个关键发现：只选影响最大的 3 个。
- 类型效率表：至少包含数量、均值曝光、新增关注、涨粉效率。
- 最大无用功：说明证据和砍掉/降频建议。
- 可复制爆款公式：用结构表达，不照搬原文。
- 下一期假设：最多 3 条，每条都能在 7 天或 30 天内验证。
- 下一步路由：节奏计划进 `/achuan-x-content-rhythm`，状态保存进 `/achuan-x-save`。

## 数据来源分层

| 来源 | 可回答的问题 | 注意 |
|---|---|---|
| X Analytics CSV | organic 内容类型、涨粉效率、曝光和互动结构 | 列名不匹配时先做字段映射 |
| Ads Manager export | campaign、ad group、creative、spend、impressions、clicks、CTR、CPC 等投放指标 | 不能和自然内容混算 |
| 商单/赞助记录 | 品牌、披露方式、交付内容、曝光、点击、付款和争议 | 同时进入 `/achuan-x-monetization-path` 检查品牌匹配和披露 |

## 复盘可信度

| 等级 | 条件 |
|---|---|
| A | CSV + 成长档案 + 上期计划 + 主页/风险状态完整 |
| B | CSV + 当前粉丝数 + 代表内容链接 |
| C | 手动数据 + 代表内容截图 |
| D | 只有主观描述，先补数据 |

## 边界

- 不承诺某个计划一定涨粉。
- 不鼓励刷互动、互推互刷、自动化批量回复或复制爆款原文。
- 不把成人擦边内容作为默认增长方向；涉及平台规则、收益、成人内容时转风险或收益模块处理。
