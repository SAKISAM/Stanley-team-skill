# 账号风险恢复与可见度异常

来源说明：本文件为开源版蒸馏规则，不依赖本地素材库。涉及平台规则、官方表单或外部检测工具时，使用前应核验最新状态。

## 来源优先级

| 优先级 | 来源 | 用法 |
|---|---|---|
| A | X 官方 Help / Rules / 表单 | 作为锁定、限制、暂停、搜索过滤和申诉的主依据 |
| B | 用户原始提示、邮件、倒计时、截图转文字 | 作为个案判断的直接证据 |
| C | 第三方 shadowban/search checker | 只作为可见度线索，不替代官方判断 |
| D | 社区经验、博客、论坛 | 只能提炼低风险自查动作，不能写成确定结论 |

## 公理

- 新号和刚解冻账号的第一目标不是冲量，而是恢复正常账号信号。
- 影子封禁检测工具只能提供线索，不能替代平台通知和实际可见度观察。
- 风险恢复要先停高风险行为，再查提示、行为记录和账号状态。
- “影子封禁”在 skill 内统一称为“可见度异常/搜索过滤/回复降权”，除非用户原话使用影子封禁。
- 账号异常处理优先级高于涨粉节奏；账号仍在限制期时，不输出高频增长计划。

## 必问材料

| 材料 | 用途 |
|---|---|
| @handle | 判断是否能公开访问、搜索或申诉对应账号 |
| 原始提示文案 | 区分锁定、受限、暂停、垃圾标签、登录锁 |
| 发生时间 | 判断倒计时、申诉窗口和恢复观察期 |
| 是否能登录 | 区分访问恢复和执行限制 |
| 是否能发帖/回复/点赞/关注/私信 | 判断限制范围 |
| 最近 72 小时行为 | 找触发因素：高频、重复、关注、第三方工具、敏感内容 |
| 最近登录环境 | 判断安全锁定、IP/设备/地区变化 |
| 是否授权第三方应用 | 排查自动化、重复发布、X Pro/Hootsuite 等连接 |
| 是否已经申诉 | 避免重复低质量申诉 |

## 高风险信号

| 行为/现象 | 风险 |
|---|---|
| 频繁切换 IP | 可能触发异常登录或信任下降 |
| 短时间大量模板式发帖/回帖 | 可能被视为机器人或垃圾互动 |
| 频繁关注/回关 | 可能触发自动化或互关风险 |
| 新号/刚解冻马上高频互动 | 可能被再次限制 |
| 内容涉黄、涉政、未标注商单 | 可能导致举报、收益暂停或可见度风险 |
| 滥用话题标签或趋势词 | 可能触发搜索过滤或低质量判断 |
| 多账号发布相似内容 | 可能触发平台操纵/搜索过滤 |
| 第三方应用批量更新账号 | 可能导致搜索过滤或自动化风险 |

## 优先级排序

| 优先级 | 场景 | 处理原则 |
|---|---|---|
| P0 | 无法登录、疑似被盗、登录尝试过多 | 先恢复控制权：等锁、改密码、检查邮箱/2FA、关闭异常第三方应用 |
| P1 | 官方提示 locked/limited/read-only | 按提示完成邮箱/手机/reCAPTCHA、删帖或等待倒计时 |
| P2 | account suspended | 准备申诉材料；先登录被暂停账号，再打开申诉入口 |
| P3 | spam label 或平台操纵提示 | 审计重复内容、自动化、高频互动、多账号相似行为 |
| P4 | 搜索不到、回复不可见、疑似影子封禁 | 先做可见度测试和 72 小时行为审计 |
| P5 | 单纯曝光下降 | 转 `/st-growth-diagnosis`，不要误判成风控 |

## 检查流程

1. 要求用户提供原始提示文案和发生时间。
2. 记录最近 72 小时行为：发帖、回复、关注、登录设备、IP/地区变化、支付/验证动作。
3. 判断是登录安全、功能受限、暂停、可见度异常还是收益暂停。
4. 先停高频和模板化动作，再按平台提示完成验证、等待或准备申诉。

## 分类处理

### P0 登录锁 / 疑似被盗

- 如果是多次登录失败，通常先等待官方提示的锁定时间，不持续重试。
- 关闭可能造成持续登录失败的第三方客户端或管理工具。
- 能登录后立即检查邮箱、电话、密码、2FA 和授权应用。
- 如果无法访问邮箱或手机号，转账号访问支持表单，不建议新开小号冒充申诉。

### P1 locked / limited / read-only

- 按页面提示完成电话、邮箱或 reCAPTCHA 验证。
- 如果系统要求删除违规帖，先记录链接、文案和原因，再按提示处理或申诉。
- 如果有倒计时，启动倒计时后保持低动作，不反复刷新、切设备或连续操作。
- 如果反复出现相同限制，进入行为风险复盘，不继续加频。

### P2 suspended

- 必问：暂停时间、官方提示/邮件全文、是否有具体政策类别、是否已申诉、是否有新增证据。
- 申诉前整理：账号用途、近期异常解释、已完成的安全动作、认为误判的证据、愿意遵守规则的整改。
- 申诉入口：https://help.x.com/en/forms/account-access/appeals
- 如果多账号登录容易串号，用独立浏览器窗口登录被暂停账号再打开表单；不要提交他人账号。
- 不建议机械重复申诉；二次申诉必须补充新证据或回应上次回执。

### P3 spam label / platform manipulation

- 审计近 7 天：重复文案、重复链接、相同标签、短时间大量回复、激进关注/取关、授权工具、多账号协同。
- 先停止自动化和模板行为，再保留少量原创、低频、高信息增量动作。
- 如果官方提供 label review，按官方入口提交复核；如果复核未移除，按只读/暂停风险处理。

### P4 可见度异常 / 搜索过滤 / 回复降权

- 先给第三方检测入口：https://x-shadowban-checker.fia-s.com/ ，并说明它只提供线索。
- 先确认不是普通内容衰减：对比同类内容、发布时间、粉丝活跃、整体平台波动。
- 手动测试：无痕或未登录状态搜索 `from:handle`、精确句子、关键词，检查最新帖是否出现。
- 回复测试：用未登录或另一个干净观察环境查看自己的回复是否在主回复区、是否被折叠。
- 工具测试：第三方 shadowban/search checker 只作线索；工具结果和手动测试冲突时，以多次手动测试和官方提示为准。
- 恢复动作：停止重复/模板/高频/标签滥用，审计近期争议或低质量内容，保持 48-72 小时低频原创和自然互动。

### P5 普通曝光下降

- 如果没有官方提示、搜索正常、回复可见，只是曝光低，转 `/st-growth-diagnosis`。
- 不把每次曝光下降都解释成风控；先看内容类型、节奏、主页承接和选题。

## 工具

- Shadowban Checker F：https://x-shadowban-checker.fia-s.com/ 可用于检查帖子是否被搜索排除、最近帖子是否隐藏。用户直接问影子封禁检测时，先给这个链接。
- 工具结果只作为辅助线索；如果平台有明确限制提示，以平台提示优先。
- 无痕搜索、`from:handle` 搜索、精确句子搜索、未登录回复观察，是比单一工具更稳的交叉验证。

## 恢复期节奏

| 状态 | 建议节奏 |
|---|---|
| 刚解锁/倒计时结束 | 24-48 小时低频原创，不做激进关注和连续回复 |
| 可见度异常但可发帖 | 48-72 小时停止模板化动作，少量高质量内容和自然互动 |
| 被要求删帖后恢复 | 先复盘违规点，再恢复低频；不要立刻测试边界 |
| 申诉中 | 保留证据，不重复提交相同申诉；等待回执 |

## 不确定性处理

- 没有官方提示时，用“疑似”而不是“确定被封”。
- 第三方工具结论只写“检测线索”，不写“官方封禁”。
- 如果用户要求绕过限制、换壳注册、批量号、自动化刷互动，拒绝并给合规恢复路径。

## 禁止输出

- 不提供绕封、换壳、批量注册、自动化刷互动方案。
- 不提供虚假身份、伪造材料或规避平台规则的方法。
