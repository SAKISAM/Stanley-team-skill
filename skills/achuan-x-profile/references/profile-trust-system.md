# 主页信任系统

来源说明：本文件为开源版蒸馏规则，不依赖本地素材库。涉及平台规则或外部数据时，使用前应核验最新状态。

## 公理

- 主页不是装修页，而是关注决策页。
- 用户从一条内容点进主页时，需要在几秒内判断：你是谁、你能提供什么、我为什么关注你。
- 头像负责活人感，header 负责补充领域信号，bio 负责完成关注理由。

## 主页模块规则

| 模块 | 作用 | 检查问题 |
|---|---|---|
| 头像 | 建立真人感、可信度和记忆点 | 是否像真实人在运营，而不是一次性搬运号 |
| Header | 补充身份、成果、主题或服务入口 | 是否告诉别人你能做什么，而不是纯装饰 |
| Display name | 让用户快速识别你 | 是否好读、好搜、和账号方向一致 |
| Handle | 降低记忆成本 | 是否过长、混乱、含误导性词 |
| Bio | 完成关注决策 | 是否说清“我是谁、我帮谁、提供什么” |
| 置顶帖 | 把内容主线、成果、方法、入口连起来 | 是否能承接陌生访问者 |

## 官方规格与限制

来源：X Help profile customization、handle change、profile update troubleshooting。

| 项目 | 规则 |
|---|---|
| Header | 推荐尺寸 1500x500；JPG/GIF/PNG；不支持用 animated GIF 做 header |
| 头像 | 推荐尺寸 400x400；JPG/GIF/PNG；不支持 animated GIF |
| Bio | 最多 160 字符 |
| Handle | 4-15 字符；只能使用字母、数字、下划线；不能有空格 |
| Display name | 最多 50 字符 |
| 置顶帖 | 用户访问主页时优先看到，可用于承接账号方向、成果、方法和入口 |
| 保存失败 | 常见原因包括假蓝勾符号、疑似恶意链接、bio 超长、尖括号或引号 |

## 主页评分卡

总分 100，只用于排序修改优先级。

| 维度 | 分值 | 判断 |
|---|---:|---|
| 身份清晰 | 20 | 5 秒内知道这个账号是谁在运营 |
| 价值清晰 | 25 | 说清能持续提供什么 |
| 信任证据 | 20 | 有经历、结果、案例、过程或作品支撑 |
| 下一步动作 | 15 | 链接或置顶帖让用户知道关注后做什么 |
| 视觉一致 | 10 | 头像/header/name/bio 指向同一方向 |
| 平台卫生 | 10 | 无假勾、危险链接、超长 bio、误导身份 |

## 主页到内容承接

来源：X Business organic best practices、X Ads glossary。

| 检查项 | 判断 |
|---|---|
| 最近内容流 | 主页最近 5-10 条内容是否持续兑现 bio 里的承诺 |
| CTA | 置顶帖或代表内容是否有清晰下一步，但不强迫购买 |
| 内容语气 | 是否 conversational、简洁、避免全大写和堆 hashtag |
| 媒体承接 | header、置顶帖、图文是否服务同一账号方向 |
| 置顶刷新 | 置顶帖不是永久招牌；当账号阶段、产品或内容主线变化时应刷新 |

## 商业入口检查

| 场景 | 处理 |
|---|---|
| bio/置顶帖放咨询、课程、社群、商单入口 | 转 `/achuan-x-monetization-path` 设计最小验证和披露边界 |
| 放跨境收款、Stripe、Wise、订阅入口 | 转 `/achuan-x-infra` 检查真实身份、地区、税务和打款条件 |
| 放联盟链接或品牌合作内容 | 提醒检查 Paid Partnership / affiliate disclosure |

## Bio 模板

不要替用户虚构身份，只用用户给出的事实填空。

| 类型 | 结构 |
|---|---|
| 过程型 | 我正在做 `{过程}`，记录 `{对象}` 如何解决 `{问题}` |
| 经验型 | `{身份/经历}`，分享 `{领域}` 的 `{方法/案例/工具}` |
| 服务型 | 帮 `{对象}` 用 `{方法}` 达成 `{结果}`，案例/入口见置顶 |
| 研究型 | 持续研究 `{主题}`，拆解 `{案例/工具/趋势}` 给 `{对象}` |

## 置顶帖结构

1. 一句话说明这个账号长期写什么。
2. 说明为什么是你写：经历、案例、项目、正在实践的过程。
3. 给 3-5 个代表性内容入口。
4. 给关注理由：关注后能持续获得什么。
5. 如果有产品/服务/社群，只放低压入口，不强卖。

## 输出时优先级

1. 先改 bio 和置顶帖，因为它们直接影响关注理由。
2. 再改头像和 header，因为它们影响第一印象和可信度。
3. 最后改 handle/display name，避免频繁改名造成识别混乱。

## 风险边界

- 不建议伪造成熟履历、虚假成果或误导性身份。
- 不把 header 做成无信息量海报；它应服务账号方向和转化。
- 不在 name 或 bio 里使用假认证符号。
- 可疑 URL 先移出 bio/website，排除保存失败和账号信任问题。
