# 素材库与内容格式分工

来源说明：本文件为开源版蒸馏规则，不依赖本地素材库。涉及平台规则或外部数据时，使用前应核验最新状态。

## 公理

- 内容系统不是“今天发什么”，而是让账号持续产生短推、长文、回复、引用和转化入口。
- 短推负责破圈，长文负责信任，回复/引用负责进入他人的流量池和建立关系。
- 素材要先结构化，AI 才能稳定拆解、改写、复用。

## 格式分工

| 格式 | 作用 | 适用场景 |
|---|---|---|
| 短推 | 测话题、破圈、制造互动 | 热点、情绪、反差、问题、轻观点 |
| 长文/thread | 建信任、立人设、沉淀方法 | 专业能力、复盘、教程、系统性观点 |
| 回复 | 进入大号流量池、建立关系 | 新号冷启动、自己流量不足 |
| 引用 | 借已验证信息表达自己的观点 | 有一定粉丝基础后放大观点 |
| 图文 | 跨语言传播、提高停留和点击 | 多国评论、教程截图、封面 |

## 官方格式约束

来源：X Help types of posts、Articles、how to post、repost/quote。

| 格式 | 官方/平台约束 | 策略含义 |
|---|---|---|
| 普通 post | 可含文本、图片、GIF、视频、链接；会出现在个人主页和关注者时间线 | 用于日常测试、观点、热点、轻量信息 |
| Longer post | 可超过普通 280 字限制，最高到 25,000 字符；需要 Premium | 用于完整方法、复盘、教程，不应替代短推测试 |
| Article | Premium/Premium+、Premium Business/Organizations 可发布；可包含图片、视频、GIF、posts、链接和格式化文本 | 更像长内容资产，适合沉淀高信任内容 |
| Thread | 多条 post 串联；引用 thread 时会显示查看整条 thread 的提示 | 适合把复杂内容拆成连续阅读 |
| Reply | 回复会出现在自己主页；对方通知可见；进入他人对话场 | 适合关系增长和流量池进入，但不是所有粉丝都会在主页时间线看到 |
| Quote post | 转发并加自己的评论，可让自己的关注者看到 | 适合借势表达判断、补充上下文和二次分发 |

## 内容支柱设计

每个账号至少保留 3 类支柱：

| 支柱 | 目标 | 常用格式 |
|---|---|---|
| 破圈内容 | 让陌生人第一次看见你 | 短推、quote、热点评论 |
| 信任内容 | 让陌生人愿意关注 | longer post、thread、Article、案例复盘 |
| 关系内容 | 让同领域的人记住你 | reply、quote、List 互动 |
| 承接内容 | 让需求明确的人知道下一步 | 置顶帖、案例帖、服务说明、社群入口 |
| 商业内容 | 合规承接品牌、联盟或赞助 | Paid Partnership、案例型商单、测评、工具推荐 |

## Organic 内容规则

来源：X Business organic best practices、campaigns 101。

| 规则 | 用法 |
|---|---|
| 简洁 | 短推只讲一个核心判断，不把长文压成一段 |
| CTA | 有链接、报名、阅读、下载时才放清晰 CTA；普通观点不硬转化 |
| 话题标签 | 不堆 hashtag；只有当标签能帮助主题聚合时才用 |
| 语气 | conversational，避免全大写、机器人腔和营销腔 |
| 媒体 | 图、视频、截图用于提升停留，不用重文字海报堆信息 |
| 创意轮换 | 同一目标至少准备 3-5 个角度或素材形态，避免单变量误判 |
| 测试复盘 | 让数据判断 copy、creative、tone 哪个有效 |

## 商业内容分层

| 类型 | 进入条件 | 必查 |
|---|---|---|
| 联盟/分佣 | 自己真实使用过，粉丝有明确需求 | affiliate disclosure、链接透明度、品牌适配 |
| 商单/赞助 | 有稳定垂直受众和互动 | Paid Partnership disclosure、禁投品类、交付指标 |
| 自有产品 | 已有服务、课程、社群或工具 | 主页入口、支付/退款/税务、用户承诺边界 |

## 选题池字段

每条选题至少记录：

| 字段 | 说明 |
|---|---|
| 目标用途 | 破圈 / 信任 / 关系 / 承接 |
| 目标读者 | 谁会因为这条内容停下来 |
| 触发来源 | 热点、案例、问题、工具、数据、个人经历 |
| 格式 | 短推、长文、thread、reply、quote、Article |
| 验证指标 | 曝光、关注、收藏、回复、私信、点击 |
| 下一步复用 | 能否拆成短推、做成长文、改成回复或置顶入口 |

## 素材库最低标准

- 10 篇可拆短推的长文。
- 5 个可模仿结构的爆款案例。
- 3 个所在赛道的长期对标账号。

## 素材源

| 来源 | 用途 | 注意 |
|---|---|---|
| X Viral Monitor | 热帖速度、可见页排行榜、Markdown 复制 | 用来找正在起量的帖子，不代表可以搬运 |

## 重组规则

- 不按原文归档内容，按账号用途归档：涨粉、信任、回复、转化、风险、复盘。
- 从长文中提取论点后，改写成短推；从爆款长文中提取结构后，套入自己的素材。
- 引用长文必须加入自己的判断，不原样转发。

## TOP 帖拆解

数据复盘后，不按原帖归档，而是拆成可复用结构：

| 拆解项 | 提问 | 进入哪里 |
|---|---|---|
| 主题 | 它解决了谁的什么问题 | 内容支柱 |
| 钩子 | 开头用了数字、反差、身份代入、时效还是利益 | 短推/长文模板 |
| 证据 | 它用了数据、案例、截图、经历还是工具 | 素材库 |
| 转粉理由 | 为什么陌生人看完会关注 | 主页/账号方向检查 |
| 可复制变量 | 哪些词可替换，哪些结构不能变 | 模板库 |

## 爆款公式写法

- 用结构表达，不复制原文：`对象 + 具体数字 + 反差发现 + 普通人收益`。
- 保留可验证信息，例如发布时间、stars、收入数字、测试时长；模糊形容词降权。
- 公式必须能服务账号方向，不能为了泛流量偏离长期主线。
- 每个公式都要配一个“适合阶段”：测试期、增长期、信任期或转化期。

## 模板库分层

| 模板层 | 作用 | 示例变量 |
|---|---|---|
| 短推测试模板 | 快速测试选题 | 话题、数字、对象、利益 |
| 长文结构模板 | 建立信任和收藏 | 钩子、背景、要点、步骤、收尾 |
| 回复模板原则 | 进入流量池 | 补充事实、案例、反问、经验 |
| 引用模板原则 | 借势表达立场 | 原帖判断、你的反差、延伸结论 |

## 30 天选题池

- 30 天计划不需要写满正文，只需要写清每天的主方向、长文题目、短推测试方向和复盘指标。
- 选题分配先看数据：高涨粉效率内容加码，低效率内容降频，未验证方向保留少量测试位。
- 每周至少留一个复盘位，把新数据回流到公式和模板库。
- 不把 Article 当成日常短推替代品；Article 用来沉淀高信任资产。
- 每周至少有一条“承接内容”，否则破圈流量无法沉淀成关注或商业线索。

## 风险边界

- 不洗稿，不造谣，不搬运未授权内容。
- 涉及热点时优先核验事实来源。
