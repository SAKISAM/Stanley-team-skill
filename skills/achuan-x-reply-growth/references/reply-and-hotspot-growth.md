# 回复与热点增长

来源说明：本文件为开源版蒸馏规则，不依赖本地素材库。涉及平台规则或外部数据时，使用前应核验最新状态。

## 公理

- 新号没有自己的流量池时，回复和引用是进入别人流量池的入口。
- 回复不是刷存在感，而是把你的判断、经验或补充放到目标人群面前。
- 热点监测工具只负责发现机会，真正转粉来自有信息增量的回复。

## 回复对象

| 对象 | 目的 | 回复方式 |
|---|---|---|
| 同赛道大号 | 借已有流量池 | 补充案例、反问、延伸观点 |
| 正在起量热帖 | 抢早期可见度 | 简短有力、不要模板化 |
| 潜在客户 | 建立信任 | 经验型、解决问题型 |
| 同阶段账号 | 建立关系 | 真诚互动，不互刷 |
| 素材源账号 | 发现话题 | 收藏、引用、二创观点 |

## 官方可见性规则

来源：X Help replies/mentions、conversations、Lists。

- 低质量或疑似 spam 回复可能被放到 “Show probable spam” 之类的低质量区。
- 帖主可以隐藏回复，被隐藏的回复仍可通过隐藏回复入口访问，但作者不会收到被隐藏通知。
- 回复会出现在自己的主页，也会通知被回复者；但不等于所有关注者都会在主页时间线看到。
- 如果希望自己的关注者也看到某条回复，官方建议使用 Repost 或 Quote post。
- X Lists 可用来建立互动名单；List 名称有长度限制，可设为 private。

## 回复质量评分

每条目标回复先用 0-2 分自评，低于 6 分不发。

| 维度 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|
| 相关性 | 可套任何帖子 | 基本相关 | 只能用于这条帖 |
| 信息增量 | 只有情绪 | 有观点 | 有案例/数据/步骤 |
| 关系价值 | 像蹭流量 | 有礼貌 | 能开启对话 |
| 风险 | 模板/攻击/争议 | 略泛 | 安全、具体、自然 |

## 互动名单结构

| List | 人群 | 使用方式 |
|---|---|---|
| A 类大号 | 同领域高流量账号 | 每天看少量，找早期高质量回复位 |
| B 类同行 | 同阶段或略高阶段账号 | 建关系、互相看见，但不互刷 |
| C 类潜在客户 | 经常表达痛点的人 | 解决问题型回复，不硬推销 |
| D 类素材源 | 新闻源、工具源、案例源 | 收藏、引用、进入选题池 |

## 社区管理 routine

来源：X Business organic best practices。

| 动作 | 用法 | 边界 |
|---|---|---|
| 关键词监听 | 监控品牌名、产品名、领域关键词和典型痛点 | 不骚扰，不批量私信 |
| 间接提及 | 搜索没有 @ 你的讨论，判断是否能补充信息 | 不强行插入无关对话 |
| 上下文检查 | 回复前看对方过往对话，避免误判需求 | 不挖隐私，不攻击 |
| 预设回答 | 对常见问题准备回答结构 | 只能做结构，必须按具体帖子改写 |
| 趋势事件 | 发现和内容主题、目标读者相关的实时讨论 | 不追所有热点，不造谣 |

## 工具使用

- 用 X Viral Monitor 等工具识别正在起量的帖子。
- 看到热帖后不要只写“学习了”“太强了”，要补充一个事实、角度、案例或反直觉判断。
- 适合收益前或自有流量不足时使用。

## 频率与安全

- 新号或刚解冻号先低频观察，避免快速连续回复。
- 不要频繁模板式回复。
- 不要频繁关注、回关。
- 不要用自动化批量回复替代真实互动。

## 回复占比诊断

- 回复占比不是越高越好；要看回复是否带来新增关注、关系增量或有效对话。
- 当回复占比超过 30%，且回复带来的新增关注很低，应怀疑原创内容被挤压。
- 当回复占比超过 40%，先做回复预算，不要继续无差别加互动。
- 如果回复很多但转粉低，优先保留高质量对象：同赛道大号、正在起量热帖、潜在客户、强素材源。

## 回复预算

| 场景 | 回复策略 | 下一步 |
|---|---|---|
| 新号无自有流量 | 少量高质量回复进入流量池 | 同步建设短推和主页 |
| 数据显示回复低效 | 降低低质量回复，保留高价值互动 | 转 `/achuan-x-content-rhythm` 重排节奏 |
| 高质量回复带来关注 | 固定互动名单和时间窗 | 转 `/achuan-x-save` 保存 routine |
| 回复后疑似受限 | 停止连续高频互动，检查风险 | 转 `/achuan-x-risk-recovery` |

## 回复复盘

| 动作 | 触发 |
|---|---|
| 保留 | 带来关注、有效对话、收藏、被高质量账号继续互动 |
| 改写 | 有互动但没有转粉，可能观点泛、缺证据或和主页承接不一致 |
| 删除/停止复用 | 被隐藏、引发无意义争吵、像模板、和内容主题无关 |
| 转内容 | 某类回复连续带来有效互动，应改成短推或长文选题 |

## 执行评分中的回复项

- 复盘时可把“回复比例达标”作为执行评分的一部分，但不能机械套用。
- 如果账号处于冷启动，回复是必要入口；如果账号已有流量，回复应更多承担关系维护和素材发现。
- 回复预算的目标不是压低互动，而是把低转粉、低信息增量的回复替换成原创内容或高质量引用。

## 路由

- 回复多但不涨粉，转 `/achuan-x-growth-diagnosis`。
- 怀疑回复导致受限，转 `/achuan-x-risk-recovery`。
- 需要选题池支持回复，转 `/achuan-x-content-system`。
