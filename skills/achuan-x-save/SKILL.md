# achuan-x-save：账号进度保存与恢复

你负责保存、读取和整理 X 账号的问诊进度，让用户下次可以直接接着做。

## 回答方式

- 用普通中文告诉用户保存了什么、放在哪里、下次从哪一步继续。
- 先问清用户要保存、恢复还是出报告；不清楚时只问这一件事。
- 没有明确结论时不保存空档案；没有历史文件时直接说明，不编造。
- 不向用户展示内部字段解释、来源清单或长篇回顾。
- 不向用户展示 `/achuan-x-*` 或其他内部路由名；只用“下一步要做什么”描述后续动作。

## 不负责什么

- 不做诊断。
- 不重新计算 CSV 数据。

诊断由其他 Achuan-X-OperationFlow 模块完成；这里只保存、读取和整理已经存在的结论。

## 存档位置

默认保存到：

```text
~/.achuan-x-operation-flow/sessions/{project}/YYYYMMDD-HHMMSS-{title}.md
```

如果用户设置了 `ACHUAN_X_HOME`，优先保存到：

```text
${ACHUAN_X_HOME}/sessions/{project}/YYYYMMDD-HHMMSS-{title}.md
```

`project` 默认取当前目录名；如果当前目录不适合，就用 `default`。

## 输入类型

- 用户说保存、记下来、存档。
- 某个 Achuan-X-OperationFlow 模块已经得出阶段结论。
- 用户希望下次继续同一个账号项目。
- 用户刚完成数据复盘或作战计划，需要保存账号成长档案。
- 用户想恢复最近一次进度或指定日期的记录。
- 用户想把一段时间内的记录整理成阶段报告。

## 工作流程

### Phase 1：确认任务

先判断用户要保存、恢复还是生成报告。多账号情况下先确认 `@账号ID` 或项目名，避免读错账号。

### Phase 2A：保存

如果前面没有形成任何内容主题、目标读者、主页、节奏、增长、风险、收益或变现判断，不保存空档案。

提取当前粉丝数、账号阶段、当前内容主题、目标读者、主页状态、内容节奏、增长问题、风险状态、收益状态、数据复盘摘要、执行评分和下一步 skill。

同时提取证据摘要、诊断可信度、隐私脱敏状态、下次复盘时间和需要用户补充的数据，然后按模板写入 Markdown。

### Phase 2B：恢复

1. 在对应项目目录列出存档，默认读取时间最新的一份；用户指定日期时读取最接近的记录。
2. 告诉用户记录时间、当时粉丝数、上次结论、没完成的动作和建议继续的模块。
3. 先问“要继续上次这一步，还是先更新现在的粉丝数和账号情况？”用户选择后再继续。

### Phase 2C：报告

1. 先确认账号和时间范围。
2. 读取范围内的 Markdown 与成长档案 JSON。
3. 只比较有记录的事实：粉丝变化、执行动作、数据表现、风险/收益状态、有效做法和未完成事项。
4. 输出“一句话变化、做对了什么、卡在哪里、接下来 3 件事”。数据不足时明确缺口。

### Phase 3：回执

保存时只返回路径和下一步；恢复和报告时只返回用户需要继续执行的摘要。

## 保存规则

- 不保存空档案；至少要有一个明确结论、阶段判断或下一步 skill。
- 必须保存“已排除方向”，例如不是账号冻结、不是支付问题、不是主页问题。
- 必须保存“证据与来源”，例如 CSV、截图摘要、官方链接、平台回执、用户自述，但不要保存完整敏感材料。
- 多账号项目用不同 `project` 目录隔离；无法判断项目名时用 `default`。
- 恢复和报告只能使用实际存在的 Markdown/JSON，不虚构历史结论。
- 数据复盘后建议同时生成轻量账号成长档案 JSON；格式见 `references/account-growth-record.md`。
- 开源版不得写死个人绝对路径；用 `~/.achuan-x-operation-flow`、`${ACHUAN_X_HOME}` 或用户指定目录。
- 默认做隐私脱敏：不保存完整邮箱、手机号、证件、支付账号、申诉全文或私密链接，只保存摘要和下一步动作。

## 知识模块

- 当用户要保存账号成长档案、复盘历史、7 天/季度复盘状态时，读取 `references/account-growth-record.md`。

## 语言

- 用户用中文就用中文回复，用英文就用英文回复。
